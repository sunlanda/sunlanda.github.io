关于github commit后首页不绿的解决方法

关于github commit后首页不绿的解决方法

楼主因为经常用公司账号提交,加上比较懒,自己的git就没怎么管,这么多灰块块,只有星星点点绿,所以就想着把用公司账号commit的找回来,也参考了官网的方法,英文好的盆友可以直接移步:
1.问题排查
2.查询当前邮箱的方法
3.替换源文件的方法

###可能出现的原因:

github登录邮箱和当前push项目使用的邮箱不匹配

使用了图形化代码提交工具(非git bash/powershell一类)

当前使用邮箱没有经过github邮箱验证(代表不是自己账号提交的)

###1.查看当前使用的邮箱或用户名:
$ git config user.email

$ git config user.name

如果不同请执行以下命令:
$ git config user.email "YouEmail@XXX.com" 
如果得出结果和github账号相同还是不绿,就不要往下看了,我解决不了

###2.如果不同,检查一下当前使用的图形化工具账户情况:

去掉全局设置(只对当前仓库生效),修改为你github的登陆邮箱,以后再使用这个账户或者图形化工具提交此仓库就会有记录.

###3.最后,在github中setting里,将当前使用的邮箱添加进来并认证

添加邮箱后,你会收到激活邮件,点击激活后,刷新自己的主页就会发现之前用此账号commit的绿色全都回来了!

###
